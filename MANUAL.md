# 2단계. 견적 신청이 "절대 누락되지 않게" 만들기 (매우 중요)

지금은 손님이 견적을 넣으면 구글 시트에만 쌓입니다. 시트를 자주 안 보면 놓칠 수 있어요.
**신청이 들어올 때마다 사장님 이메일로 즉시 알림이 오도록** 바꾸겠습니다. (사진도 여러 장 저장)

### 따라하기
1. 구글에서 본인 계정으로 로그인 → 기존에 만들어둔 **Apps Script**(견적이 저장되는 스크립트)로 이동
   - 모르시면: 견적이 쌓이는 **구글 시트 열기 → 상단 메뉴 [확장 프로그램] → [Apps Script]**
2. 기존 `doPost` 코드를 지우고, 아래 코드를 **그대로 붙여넣기** 하세요.
   (이메일 주소는 사장님 주소로 이미 적어뒀습니다)

```javascript
function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    var sheet = ss.getSheets()[0];
    var region = (data.province || "") + " " + (data.city || "");
    var when = data.dateNone ? "날짜무관" : (data.date || "");
    sheet.appendRow([new Date(), data.service, region, data.content, when, data.phone, data.photoCount || 0, data.agreedPrivacy]);

    // 신청 즉시 사장님 이메일로 알림 보내기 (절대 누락 방지)
    MailApp.sendEmail({
      to: "jung22459369@gmail.com",
      subject: "📋 새 견적신청: " + data.service + " / " + region + " / " + data.phone,
      body:
        "■ 서비스: " + data.service + "\n" +
        "■ 지역: " + region + "\n" +
        "■ 내용: " + data.content + "\n" +
        "■ 희망일: " + when + "\n" +
        "■ 연락처: " + data.phone + "\n" +
        "■ 사진수: " + (data.photoCount || 0) + "장\n" +
        "■ 개인정보 동의: " + data.agreedPrivacy
    });

    return ContentService.createTextOutput(JSON.stringify({ result: "ok" }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({ result: "error", msg: String(err) }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
```

3. 상단 **💾 저장** → 오른쪽 위 **[배포] → [배포 관리]** 클릭
4. 연필(수정) 아이콘 → **버전: "새 버전"** 선택 → **[배포]** 클릭
   - 권한 요청이 뜨면 본인 계정으로 **허용**하세요. (이메일 보내려면 권한이 필요합니다)
5. 끝! 이제 견적이 들어오면 **사장님 Gmail로 바로 알림**이 옵니다.

> ⚠️ 새 스크립트 주소(URL)가 바뀌지 않았다면 사이트는 그대로 작동합니다.
> 만약 "새 배포"로 **주소가 바뀌었다면**, 그 새 주소를 저에게 알려주세요. 사이트 코드에 갈아끼워 드리겠습니다.

# 3단계. 구글 검색에 등록하기 (구글 서치콘솔)

검색에 잡히려면 구글에게 "이 사이트가 있어요" 라고 알려줘야 합니다.

1. `search.google.com/search-console` 접속 → 본인 구글 계정 로그인
2. 왼쪽 위 **[속성 추가]** → **"URL 접두어"** 선택 → 주소 입력:
   `https://jungjin20020506.github.io/dadsun-furniture/`
3. 소유권 확인 방법 중 **"HTML 태그"** 선택 → `<meta name="google-site-verification" content="..." />` 코드가 나옵니다.
4. 그 **content 값(긴 문자열)**을 복사 → 저에게 주시면 제가 사이트에 넣어드립니다.
   - (직접 하려면: `index.html` 위쪽의 `<!-- google-site-verification ... -->` 주석을 찾아 코드로 교체)
5. 넣은 뒤 1단계처럼 반영(merge)하고 → 서치콘솔에서 **[확인]** 클릭
6. 확인되면 왼쪽 메뉴 **[Sitemaps]** → 칸에 `sitemap.xml` 입력 → **[제출]**

> 효과: 며칠~몇 주 안에 구글 검색에 사이트가 노출되기 시작합니다.

# 10단계 (선택, 효과 큼). 자체 도메인 연결

지금 주소 `jungjin20020506.github.io/...` 는 외우기 어렵고 신뢰도가 낮습니다.
`아빠와아들가구.com` 같은 **짧은 주소**를 쓰면 명함·당근·카톡에 적기 좋습니다.

1. 가비아(gabia.com) / 후이즈 등에서 도메인 구매 (연 1~2만원대)
2. GitHub 저장소 → **[Settings] → [Pages] → "Custom domain"** 에 구매한 도메인 입력 → 저장
3. 도메인 회사 관리페이지에서 DNS 설정:
   - `A` 레코드 4개: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - 또는 `CNAME` 레코드: `jungjin20020506.github.io`
4. GitHub Pages에서 **"Enforce HTTPS"** 체크
5. 도메인이 정해지면 저에게 알려주세요 — 코드 안의 주소들(canonical, OG, sitemap 등)을 새 도메인으로 한 번에 바꿔드립니다.
